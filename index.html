ProofTable model presentation

Table cells as evidence-backed claims.

ProofTable is a manifest-first model for high-stakes tables. It keeps the table readable for humans, while selected cells resolve to structured claim objects that software, LLMs, and agent tools can inspect, validate, and reuse.

The model is useful when a value should not travel without its source, formula, inputs, caveat, freshness, and safe-use rule. It verifies traceability and consistency. It does not prove that the original source is true.

A visible cell shows the value. The manifest explains the claim.

5What a claim object contains

The manifest is structured data. It is not just a note next to the table. It is the object a validator can inspect.

6Validation severity model

ProofTable separates package failures from evidence cautions. A caution does not mean the claim is unusable. It means the caveat must travel with the claim.

passClaim verifies.
cautionUsable with caveat.
warnNon-blocking issue.
failDo not reuse.
unknownInsufficient evidence.
✓ A valid claim can still be caveated.
! Caveats are part of the claim, not side notes.
× Formula mismatch should fail validation.

7Why the model works

Tables already carry meaning through row and column context. ProofTable adds an evidence dimension to selected cells.

ModelWhat the cell means
Normal table cell = visible value + row context + column context
ProofTable cell = visible value + row context + column context + evidence object

8When to use it

ProofTable is not for every table. It is for tables where reuse, citation, audit, or interpretation depends on evidence.

AI benchmark reports clinical evidence scientific tables financial assumptions compliance matrices engineering tests

Do not use it for ordinary layout tables, simple content grids, or low-stakes values where source, formula, caveat, or audit trail does not change interpretation.

9Agent-table idea starters

ProofTable is one expression of a broader pattern: tables can become structured working surfaces for humans and agents, not just rendered output.

ProofTable Cells resolve to source-backed claims with validation metadata.
Agent Table Rows include allowed actions, blocked actions, owners, and tool-use instructions.
Evidence Grid Clinical, scientific, or compliance claims carry uncertainty and audit context.
Decision Table Cells carry thresholds, rules, caveats, and next-step recommendations.

The common idea: a table should be readable by a person and operationally useful to software without flattening meaning.
